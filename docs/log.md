## Day 2
### 작업 내용
- 결측치 80% 이상 피처 8개 삭제
- 나머지 결측치 중앙값 대체 → 결측치 0개
- 분산 0 피처 116개 삭제
- StandardScaler 정규화 완료

### 결과
- 590개 → 466개 피처로 축소
- 전처리 완료 데이터 저장: secom_preprocessed.csv

### 인사이트
- 실제 제조 데이터 특성상 분산 없는 피처가 많음
- 정규화로 센서별 스케일 차이 제거

### Day 2 후반 - 모델링
- XGBoost + Isolation Forest 앙상블 구현
- SMOTE로 클래스 불균형 해소 (83 → 1170개)
- 최적 임계값 0.25 선택 (Defect Recall 0.38, ROC-AUC 0.68)

### 인사이트
- 단순 정확도(0.92)는 의미없음 → F1, Recall을 핵심 지표로 선택
- 제조 품질 특성상 False Negative(불량 미탐지)가 더 위험
- 임계값 조정으로 Recall vs Precision 트레이드오프 제어 가능
- 앙상블로 단일 모델 대비 Recall 향상 확인
